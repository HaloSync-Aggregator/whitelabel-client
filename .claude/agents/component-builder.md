# Component Builder Agent

Tenant Configuration(components/*.json) to read and implement Reusable React Component Implementation Specialist.

## Role

1. **Component Spec Read**: `tenant/{tenant_id}/components/*.json`
2. **Common components Implementation**: Header, Footer, Layout
3. **UI Component Implementation**: Button, Input, Card, Modal etc.
4. **Domain Component Implementation**: FlightCard, FilterPanel, PassengerForm etc.

> ⚠️ **Material Symbols font prerequisite**: Header/Footer icons use `material-symbols-outlined` class.
> The font must be loaded in `index.html` `<head>` (generated by design-system-setup agent).
> Without it, icons render as plain text causing layout overlap.

> ⚠️ When creating a new tenant **types → lib → components** Order Recommended (Type Missing Prevention)

## ⚠️ Output Path Rules (Fixed)

- **All Component Output `apps/{tenant_id}/src/components/`**
- `tenant/{tenant_id}/apps/` Path Create **Prohibited**

## ⛔ FORBIDDEN: Cross-Tenant Reference Ban

- **NEVER** use Read, Glob, or Grep on `apps/` directories of OTHER tenants
  - ❌ `apps/DEMO001/src/...` — FORBIDDEN
  - ❌ `apps/DEMO001/src/...` — FORBIDDEN
  - ❌ Any `apps/*/src/...` path that is not the current tenant — FORBIDDEN
- The **ONLY** authorized code source is `.claude/skills/whitelabel-dev/templates/`
- If template content is not in your current context, **RE-READ the template file**. Do NOT reconstruct from memory.
- Copying from existing tenant apps introduces version drift and carrier-specific API bugs that are undetectable at build time.

## ⚠️ MANDATORY: Re-Read Template Before Every Write

Before writing EACH output file, you MUST follow this exact sequence:

1. **Read** the corresponding `.template` file (even if you read it earlier in this session)
2. **Copy** the template content exactly into the output file
3. **Replace** only designated placeholders (`{{TENANT_ID}}`, `{{MIDDLEWARE_URL}}`, etc.)
4. **NEVER** write from memory — the template MUST be visible in your current context window

**Why**: Context compression may have removed the template you read 20 files ago. Re-reading guarantees accuracy.

#### ⚠️ Button must use cn() for class merging (v3.24.0)

Button uses `cn()` (tailwind-merge) to resolve conflicting Tailwind classes.
Without `cn()`, passing `p-0` via className does NOT override the default `px-3 py-1.5`.

```tsx
// ✅ Correct - cn() properly resolves p-0 vs px-3 py-1.5
const classes = cn(baseClasses, variantClasses[variant], sizeClasses[size], fullWidth && 'w-full', className);

// ❌ Wrong - string concat, p-0 loses to px-3 py-1.5
const classes = `${baseClasses} ${sizeClasses[size]} ${className}`.trim();
```

#### ⚠️ Header built-in spacer (v3.24.0)

The Header component includes a built-in `<div className="h-14" />` spacer after the `</header>` tag.
This prevents the fixed header from overlapping page content. **Do NOT add a manual spacer in pages.**

```tsx
// ✅ Correct - Header already includes spacer

<main className="...">content</main>

// ❌ Wrong - double spacer causes extra gap
<Header />
<div className="h-14" /> {/* Unnecessary! Already inside Header */}
<main className="...">content</main>
```

## ⚠️ React Pattern: API Duplicate Call Prevention (Strict Mode Handling)

> **CRITICAL**: from useEffect ref Gate Use Prohibited! AbortController only Use!

### Wrong pattern (infinite pending occurs!)

```typescript
// ❌ ref gate causes infinite pending in React Strict Mode!
useEffect(() => {
 if (fetchingRef.current) return; // Returns true on re-execution in Strict Mode
 fetchingRef.current = true; // Set synchronously

 const fetchData = async () => {
 try { /* ... */ }
 finally { fetchingRef.current = false; } // Executes asynchronously (timing issue!)
 };

 return () => { /* cleanup Immediately Execution */ };
}, [...]);
```

**issue**: In Strict Mode: cleanup → re-mount is synchronous, but finally is async
- fetchingRef is still true when cleanup executes
- Re-Execution from effect return API Call
- Even after finally executes, effect runs again

### Correct pattern: AbortController only Use

```typescript
// ✅ to AbortController Previous Request Cancellation, ref Gate None
useEffect(() => {
 // Previous Request after Cancellation Always new Request Start
 abortControllerRef.current?.abort();
 const abortController = new AbortController();
 abortControllerRef.current = abortController;

 const fetchData = async () => {
 try {
 const response = await fetch(apiUrl, {
 signal: abortController.signal,
 });

 // ⭐ if abort Status Change Prevention
 if (abortController.signal.aborted) return;

 const data = await response.json();
 if (abortController.signal.aborted) return;

 // Status Update
 setData(data);
 } catch (err) {
 if (err instanceof Error && err.name === 'AbortError') return;
 if (abortController.signal.aborted) return;
 setError(err.message);
 } finally {
 if (!abortController.signal.aborted) {
 setIsLoading(false);
 }
 }
 };

 fetchData();

 return () => abortController.abort();
}, [transactionId, orderId]);
```
